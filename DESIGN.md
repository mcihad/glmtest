# DESIGN SYSTEM ‚Äî KentOS Console Template

> This document is the **single source of truth** for the design system. It is
> written so that the system can be rebuilt **1:1** from this file alone, by a
> human or another agent. Every token, dimension, color, and component standard
> is specified with its exact value and the file where it lives.
>
> Golden rule: **never hardcode a value that a token can express.** Colors,
> spacing, radius, fonts, shadows, and layout geometry all flow from CSS custom
> properties defined in `src/index.css` and overridden at runtime by the theme
> engine. If you find yourself typing `#fff`, `16px`, or `rounded-[10px]`, stop ‚Äî
> there is a token for it.

> **üìÅ Repository layout.** This is now a monorepo. The frontend lives in
> **`frontend/`** and the Go backend in **`backend/`**. Unless stated otherwise,
> every path in this document (e.g. `src/index.css`, `src/routes`) is **relative
> to `frontend/`** ‚Äî i.e. the real path is `frontend/src/index.css`. The compiled
> SPA is emitted into `backend/static/` and embedded into the Go binary
> (`go:embed`) so the whole app ships as a single executable. Build/run via the
> root **`Makefile`** (`make build`, `make run`; the backend serves on `:5800`).
> See **`AGENTS.md`** for the full system architecture.
>
> **Config:** all runtime config lives in the root **`.env`** (copy from
> `.env.example`). Vite reads it too (`envDir`), so frontend env vars must be
> `VITE_`-prefixed ‚Äî e.g. `VITE_API_BASE_URL` (default `/api/v1`) for API calls.
> The app's identity (name = Keycloak client, icon, roles, public `address`, ‚Ä¶)
> is declared in **`backend/internal/module/kentos.module.json`** and served at
> `/api/v1/<name>/metadata`.

| Toasts         | `sonner`                                                         |

| Path alias     | `@/*` ‚Üí `src/*` (vite + tsconfig)                                |

### Code conventions

- **`cn()`** (`src/lib/utils.ts`) merges class names with Tailwind-aware conflict
  resolution (`clsx` + `tailwind-merge`). Always use it instead of template
  strings so later classes win predictably.
- **`data-slot`** ‚Äî every UI primitive sets a `data-slot="<name>"` attribute on
  its root for styling hooks and testing. Preserve it when editing.
- **Component API shape** ‚Äî primitives are plain functions taking
  `React.ComponentProps<...>` and spreading `{...props}`. No `forwardRef` (React
  19 passes `ref` as a prop). Keep this shape for new primitives.
- **Variants** ‚Äî use `class-variance-authority` (`cva`) for multi-variant
  components (see `button.tsx`, `badge.tsx`, `toggle.tsx`).
- **Icon size** ‚Äî default `size-4` (16px) inside buttons/menus; `size-[1.15rem]`
  (~18px) for standalone app-bar/icon-rail actions; `size-5` (20px) for brand /
  feature marks.

## 3. The Token System

Tokens live in **`src/index.css`** and are organized into four sections:

1. **Configurable tokens** (`:root`) ‚Äî the raw, overridable values.
2. **Theme mapping** (`@theme inline`) ‚Äî exposes tokens to Tailwind utilities.
3. **Base layer** ‚Äî global element defaults.
4. **Utilities** ‚Äî helper classes.

The theme engine (Section 5) overrides section 1 at runtime via an injected
`<style id="app-theme">`. Tailwind utilities consume the result automatically.

### 3.1 Typography tokens

| Token                | Default                                          | Notes |
| -------------------- | ------------------------------------------------ | ----- |
| `--font-sans`        | `'Inter', ui-sans-serif, system-ui, ‚Ä¶`           | body / UI |
| `--font-heading`     | = `--font-sans`                                   | headings |
| `--font-mono`        | `ui-monospace, 'JetBrains Mono', ‚Ä¶`              | code, kbd |
| `--font-size-base`   | `16px`                                            | root font size (`html`) |
| `--scale`            | `1`                                               | global type/zoom multiplier |
| `--text-2xs`         | `0.6875rem` (11px)                                | √ó `--scale` |
| `--text-xs`          | `0.75rem` (12px)                                  | √ó `--scale` |
| `--text-sm`          | `0.875rem` (14px)                                 | √ó `--scale` |
| `--text-base`        | `1rem` (16px)                                     | √ó `--scale` |
| `--text-lg`          | `1.125rem` (18px)                                 | √ó `--scale` |
| `--text-xl`          | `1.25rem` (20px)                                  | √ó `--scale` |
| `--text-2xl`         | `1.5rem` (24px)                                   | √ó `--scale` |
| `--text-3xl`         | `1.875rem` (30px)                                 | √ó `--scale` |
| `--text-4xl`         | `2.25rem` (36px)                                  | √ó `--scale` |
| `--leading-tight`    | `1.2`                                             | headings |
| `--leading-normal`   | `1.5`                                             | body |
| `--leading-relaxed`  | `1.65`                                            | long-form |
| `--tracking-tight`   | `-0.01em`                                         | headings |
| `--tracking-wide`    | `0.02em`                                          | uppercase labels |
| `--font-weight-*`    | normal 400 / medium 500 / semibold 600 / bold 700 | |

The ramp is wired into Tailwind in `@theme inline` as
`--text-* : calc(var(--text-*) * var(--scale))`, so `text-sm`, `text-2xl`, etc.
all scale together when `--scale` changes. `text-2xs` is a **custom** step added
by this system ‚Äî use it for meta text (timestamps, captions, uppercase group
labels).

### 3.2 Spacing

| Token       | Default            | Notes |
| ----------- | ------------------ | ----- |
| `--spacing` | `0.25rem` (4px)    | Tailwind v4's base unit. Every `p-*`, `gap-*`, `m-*`, `size-*`, `w-*` is `calc(--spacing √ó n)`. |

Lowering `--spacing` makes the **entire UI denser**; raising it makes it roomier.
The customizer exposes this as "Spacing grid" (clamped 0.20rem‚Äì0.32rem).

**Spacing usage scale (4px grid):** prefer `1`(4) `1.5`(6) `2`(8) `2.5`(10)
`3`(12) `4`(16) `5`(20) `6`(24) `8`(32). Page gutters use the layout tokens below.

### 3.3 Radius ramp

Base token `--radius` = `0.625rem` (10px). The ramp is derived in `@theme inline`:

| Utility       | Formula                  | ~px @ default |
| ------------- | ------------------------ | ------------- |
| `rounded-xs`  | `--radius ‚àí 6px`         | 4px           |
| `rounded-sm`  | `--radius ‚àí 4px`         | 6px           |
| `rounded-md`  | `--radius ‚àí 2px`         | 8px           |
| `rounded-lg`  | `--radius`               | 10px          |
| `rounded-xl`  | `--radius + 4px`         | 14px          |
| `rounded-2xl` | `--radius + 8px`         | 18px          |

Conventions: inputs/buttons/menu-items `rounded-md`; cards/popovers/dialogs
`rounded-xl`; pills/avatars `rounded-full`; the AI dock `rounded-2xl`.

### 3.4 Shadows (elevation ramp)

Shadows are built from `--shadow-color` (light: `240 5% 12%`, dark: `0 0% 0%`)
and exposed as `shadow-xs ‚Ä¶ shadow-2xl`.

| Utility       | Role |
| ------------- | ---- |
| `shadow-xs`   | inputs, buttons, switches (subtle) |
| `shadow-sm`   | cards, resting surfaces |
| `shadow-md`   | dropdowns, small popovers, map controls |
| `shadow-lg`   | popovers, command palette, sidebar fly-outs |
| `shadow-xl`   | dialogs, sheets |
| `shadow-2xl`  | AI dock (highest floating surface) |

The theme engine **regenerates all six** from a single strength multiplier
(`none 0`, `soft 0.6`, `default 1`, `strong 1.7`) ‚Äî see `generate-css.ts`.
Strength `none` yields a flat, shadowless design.

### 3.5 Layout geometry tokens

| Token                            | Default     | Used by |
| -------------------------------- | ----------- | ------- |
| `--app-sidebar-width`            | `17rem`     | expanded sidebar (`w-sidebar`) |
| `--app-sidebar-width-collapsed`  | `3.5rem`    | icon rail (`w-sidebar-collapsed`) |
| `--app-appbar-height`            | `3.5rem`    | app bar + sidebar header (`h-appbar`) |
| `--app-footer-height`            | `2.5rem`    | **shared** by page footer AND sidebar footer (`h-footer`) ‚Äî raise for a taller footer |
| `--app-page-padding-x`           | `1rem`      | PageWrapper horizontal gutter (intentionally tight) |
| `--app-page-padding-y`           | `0.875rem`  | PageWrapper vertical gutter |

These are surfaced as Tailwind utilities via `@theme inline`:
`--spacing-sidebar`, `--spacing-sidebar-collapsed`, `--spacing-appbar`,

### 3.6 Motion tokens

| Token               | Default                       | Role |
| ------------------- | ----------------------------- | ---- |
| `--duration-fast`   | `120ms`                       | hovers, small toggles |
| `--duration-normal` | `200ms`                       | theme cross-fades, most transitions |

light/dark pairs and accent presets stay balanced.

## 5. Theme Engine

The engine turns a serializable `ThemeConfig` into live CSS. Read these four
files together: `types.ts`, `presets.ts`, `generate-css.ts`, `theme-provider.tsx`.

### 5.1 `ThemeConfig` (what a theme *is*)

```ts
interface ThemeConfig {
  mode: 'light' | 'dark' | 'system'  // light / dark / follow OS
  preset: string                     // COLOR_PRESETS id
  radius: number                     // rem (slider 0‚Äì1.5)
  scale: number                      // multiplier (slider 0.85‚Äì1.25)
  fontSize: number                   // px (slider 13‚Äì18)
  spacing: number                    // rem (slider 0.20‚Äì0.32)
  fontSans: string                   // font stack
  fontHeading: string                // font stack
  fontMono: string                   // font stack
  shadow: 'none' | 'soft' | 'default' | 'strong'
}
```

`DEFAULT_THEME` (in `presets.ts`) is the factory default and the merge base for
loading older saved configs.

### 5.2 How it applies (`theme-provider.tsx`)

1. **Load** config from `localStorage["app-theme"]`, merged over `DEFAULT_THEME`.
2. **Generate CSS** via `generateThemeCss(config)` ‚Üí
   ```css
   :root { --radius:‚Ä¶; --scale:‚Ä¶; --font-size-base:‚Ä¶px; --spacing:‚Ä¶rem;
           --font-sans:‚Ä¶; --font-heading:‚Ä¶; --font-mono:‚Ä¶;
           --shadow-xs‚Ä¶2xl:‚Ä¶;                /* scaled by strength */
           --primary:‚Ä¶; --accent:‚Ä¶; --ring:‚Ä¶; /* preset LIGHT palette */ }
   .dark { --primary:‚Ä¶; --accent:‚Ä¶; --ring:‚Ä¶; /* preset DARK palette */ }
   ```
   ‚Ä¶and write it into a `<style id="app-theme">` in `<head>` (created once,
   updated on every change via `useLayoutEffect`).
3. **Mode** ‚Üí toggle `.dark` class + `color-scheme` on `<html>`. `system` mode
   subscribes to `prefers-color-scheme` and updates live.
4. **Persist** the config back to `localStorage` on every change.

Because scalar tokens are emitted on `:root` and color tokens are split across
`:root` / `.dark`, **both modes remain fully themeable** ‚Äî the same config drives
light and dark simultaneously.

### 5.3 Dark mode strategy

- `.dark` class on `<html>`, declared as a Tailwind variant in `index.css`:
  `@custom-variant dark (&:is(.dark *));` ‚Äî enabling `dark:` utilities.
- **No-flash bootstrap**: an inline script in `index.html` reads the saved config
  before paint and sets the `.dark` class, `color-scheme`, and `--font-size-base`
  so there is no flash of the wrong theme on first load.

### 5.4 Using the theme in code

```ts
const { config, mode, resolvedMode, setConfig, setMode, toggleMode, reset } =
  useTheme()
setConfig({ preset: 'blue', radius: 0.75 }) // patch + persist
```

`resolvedMode` is always `'light' | 'dark'` (system resolved); use it when you
need the *actual* applied mode (e.g. Sonner theme).

## 11. Page Primitives (`page.tsx`)

Exports: `PageWrapper`, `PageHeader`, `PageActions`, `PageFooter`,
`PageFooterStat` (the last two documented in Section 12).

### 11.1 `PageWrapper`
- `padded` (default `true`): **fills the full width** of the content area with
  only the page-padding gutters ‚Äî `mx-auto w-full px-[--app-page-padding-x]
  py-[--app-page-padding-y]`. No max-width cap by default, so content is never
  left floating with large side gaps on wide screens. Gutters are kept tight
  (1rem / 0.875rem) on purpose ‚Äî adjust the tokens, not per-page padding.
- To **cap + center** a page for readability (forms, prose), pass a `max-w-*`
  via `className` (e.g. `<PageWrapper className="max-w-3xl">`); `mx-auto` is
  retained so it centers. `--app-content-max-width` (96rem) is available as a
  convenience cap if you want a global maximum.
- `padded={false}`: `relative h-full w-full`, **no gutters**, fills the content
  area. Use for maps/canvases/editors (see `routes/map.tsx`, which absolutely
  positions its surface with `absolute inset-0`).

### 11.2 `PageHeader`
- Title row: `H1` `text-2xl font-semibold tracking-tight` (truncates) + optional
  `description` (`text-sm text-muted-foreground`) on the left; `actions` slot on
  the right (`flex shrink-0 gap-2`). Bottom margin `mb-6`.
- `actions` typically holds a secondary `Button variant="outline"` + a primary
  `Button` and/or a `DropdownMenu` (see `routes/index.tsx`).

### 11.3 `PageActions`
- Thin `flex items-center gap-2` wrapper for grouping header controls.

## 12. Footer (`footer.tsx` + `page.tsx`)

`AppFooter` ‚Äî `h-footer` (`--app-footer-height`, default `2.5rem`, **shared with

the shell outside the scroll area, so it is always visible (fixed-feel). **Raise
`--app-footer-height`** (token or theme) for a taller footer.

### 12.1 Externally-managed content
The footer's content is **controlled by the page**, not hardcoded:

- `AppFooter` renders a portal target (`<div ref={setFooterSlot} class="contents">`)
  and a **default status strip** (status dot + version ¬∑ Privacy/Terms/¬©) that
  shows only when no page has taken over (`!pageFooterActive`).
- A page takes over by rendering **`<PageFooter>`** (from `page.tsx`). It portals
  its children into the footer slot and flips `pageFooterActive` while mounted
  (via `useLayout()` state ‚Äî no render loop, unlike passing JSX through state).
- **`<PageFooterStat label value />`** is a compact label/value pair for thin
  stat strips (table totals, selection counts, page stats, bulk-action bars).

```tsx
// inside any page
<PageFooter>
  <PageFooterStat label="Rows" value="1,204" />
  <PageFooterStat label="Selected" value="12" />
  <PageFooterStat label="Updated" value="just now" className="ml-auto" />
</PageFooter>
```

Live example: `routes/analytics.tsx` renders a sessions/duration/bounce strip.
Layout-context fields backing this: `footerSlot`, `setFooterSlot`,
`pageFooterActive`, `setPageFooterActive`.

## 14. Component Standards (`components/ui/*`)

Universal rules baked into every primitive:

- **Focus ring** ‚Äî interactive elements use
  `focus-visible:ring-[3px] focus-visible:ring-ring/50` (+ `focus-visible:border-ring`
  on bordered inputs). Never remove focus styles.
- **Invalid state** ‚Äî form controls support
  `aria-invalid:border-destructive aria-invalid:ring-destructive/30`.
- **Disabled** ‚Äî `disabled:opacity-50 disabled:pointer-events-none`
  (`cursor-not-allowed` on inputs).
- **Overlays** (dialog/sheet/popover/dropdown/select/tooltip/command) animate
  with `data-[state=open]:animate-in fade-in-0 zoom-in-95` /
  `data-[state=closed]:animate-out ‚Ä¶` from `tw-animate-css`, and portal to
  `<body>`.

### 14.1 Button (`button.tsx`)
Variants: `default` (primary), `secondary`, `outline`, `ghost`, `link`,
`destructive`. Sizes: `default` (h-9), `sm` (h-8), `lg` (h-10), `icon` (size-9),
`icon-sm` (size-8), `icon-lg` (size-10). `asChild` renders as a Radix `Slot`
(used to make `Link`s look like buttons). Auto-sizes child SVGs to `size-4`.

### 14.2 Badge (`badge.tsx`)
Variants: `default`, `secondary`, `success`, `warning`, `info`, `destructive`,
`outline`. `rounded-md px-2 py-0.5 text-xs`, auto SVG `size-3`. Used for nav
counts, deltas, statuses.

### 14.3 Card (`card.tsx`)
`rounded-xl border bg-card shadow-sm py-6`, slots: `CardHeader` (grid that
supports a right-aligned `CardAction`), `CardTitle`, `CardDescription`,
`CardContent` (`px-6`), `CardFooter`. Default vertical rhythm `gap-6`.

### 14.4 Dialog (`dialog.tsx`) ‚Äî **title/header standard**
- Overlay: `bg-black/50 backdrop-blur-[1px]`, fade.
- Content: centered, `max-w-lg rounded-xl border bg-popover p-6 shadow-xl`, zoom.
- `DialogHeader` = `flex flex-col gap-1.5` (text-center on mobile, left ‚â•sm).
- `DialogTitle` = **`text-lg leading-none font-semibold`**.
- `DialogDescription` = `text-sm text-muted-foreground`.
- `DialogFooter` = `flex-col-reverse ‚Üí sm:flex-row sm:justify-end gap-2`.
- Auto close "‚úï" top-right (`showCloseButton`, default on).

### 14.5 Sheet (`sheet.tsx`)
Side drawer (`left|right|top|bottom`), `bg-popover shadow-xl`, slides per side.
`SheetHeader` padded `p-5`; **`SheetTitle` = `text-base font-semibold`**;
`SheetDescription` = `text-sm text-muted-foreground`. Used by mobile sidebar and
the theme customizer.

### 14.6 Accordion (`accordion.tsx`) ‚Äî **header/title standard**
- Items separated by `border-b` (last none).
- **Trigger** = `py-4 text-sm font-medium`, full-width, `hover:underline`, with a
  trailing `ChevronDown` that rotates 180¬∞ when open
  (`[&[data-state=open]>svg]:rotate-180`).
- Content animates with `animate-accordion-up/down`; inner padding `pb-4`.

### 14.7 Tabs (`tabs.tsx`) ‚Äî **tab style standard**
- `TabsList` = `h-9 rounded-lg bg-muted p-1` (segmented "pill track").
- `TabsTrigger` = `h-7 rounded-md text-sm font-medium`; active =
  **`data-[state=active]:bg-background data-[state=active]:shadow-sm`** (raised
  pill). Triggers can hold a leading icon.

### 14.8 Dropdown Menu (`dropdown-menu.tsx`)
`min-w-8rem rounded-lg border bg-popover p-1 shadow-lg`. Items `rounded-md
px-2 py-1.5 text-sm`, `focus:bg-accent`; `variant="destructive"` tints red.
Supports `Label`, `Separator`, `Shortcut` (right-aligned `text-xs tracking-widest
text-muted-foreground`), `CheckboxItem`, `RadioItem`, and `Sub` menus. Leading
icons auto `size-4` and `text-muted-foreground`.

### 14.9 Tooltip (`tooltip.tsx`)
`bg-foreground text-background text-xs rounded-md px-2.5 py-1` with an arrow.
Default `delayDuration` 200ms; provider is mounted once in `main.tsx`.

### 14.10 Inputs & friends
- `Input`/`Textarea`: `h-9 rounded-md border-input bg-transparent text-sm
  shadow-xs`, standard focus/invalid rings.
- `Select`: trigger matches input height; content is a portalled popover; checked
  item shows a right-aligned check.
- `Switch`: `h-5 w-9` track, `size-4` thumb, `data-[state=checked]:bg-primary`.
- `Checkbox`: `size-4 rounded-[4px]`, checked = `bg-primary`.
- `Slider`: `h-1.5` track, `bg-primary` range, `size-4` thumb with hover ring.
- `Progress`: `h-2 rounded-full bg-secondary`, `bg-primary` indicator.
- `Label`: `text-sm font-medium`, dims with disabled peers.
- `Kbd`: `h-5 rounded border bg-muted px-1.5 font-mono text-2xs` keycap.

### 14.11 Drawer (`drawer.tsx`, built on `vaul`)
Swipeable sheet with a drag handle. Direction-aware via
`data-[vaul-drawer-direction=‚Ä¶]`: bottom (default) ‚Üí
`rounded-t-2xl border-t max-h-[82vh]` + a centered `h-1.5 w-12` grab handle;
left/right ‚Üí `w-3/4 sm:max-w-sm`. Surface `bg-popover shadow-2xl`, overlay
`bg-black/50 backdrop-blur-[1px]`. Slots: `DrawerTrigger`, `DrawerContent`,
`DrawerHeader`/`DrawerFooter` (`p-5`), `DrawerTitle` (`text-base font-semibold`),
`DrawerDescription`, `DrawerClose`. Use for filters/quick forms on touch.
(Distinct from `Sheet`, which is a non-draggable Radix dialog drawer.)

### 14.12 Context menu (`context-menu.tsx`, Radix)
Right-click menu. Visual language is **identical to Dropdown Menu** (`min-w-10rem
rounded-lg border bg-popover p-1 shadow-lg`, items `rounded-md px-2 py-1.5
text-sm focus:bg-accent`, `variant="destructive"`). Supports `Label`,
`Separator`, `Shortcut`, `CheckboxItem`, `RadioGroup`/`RadioItem`, and `Sub`
menus. Wrap any region in `<ContextMenuTrigger asChild>`.

### 14.13 Table (`table.tsx`)
Semantic `<table>` wrapped in an overflow-x container. `TableHead` =
`h-10 px-3 text-xs font-medium text-muted-foreground`; `TableCell` = `p-3`;
`TableRow` = `border-b hover:bg-muted/50 data-[state=selected]:bg-accent` (set
`data-state="selected"` for row selection); `TableFooter` =
`border-t bg-muted/50 font-medium` for totals. Pair with `Checkbox` for select,
`Badge` for status, and a ghost `icon-sm` button for row menus.

### 14.14 Tree (`tree.tsx`)
Generic file-explorer tree. Data-driven by `TreeNode { id, label, icon?, badge?,
children? }`. Uncontrolled expansion (`defaultExpanded` ids), single selection
(`defaultSelected` + `onSelect`). Rows are `h-8 rounded-md`, depth indent
`paddingLeft = depth*1.1 + 0.5rem`, a `ChevronRight` that rotates 90¬∞ when open
(hidden on leaves), folders use `Folder`/`FolderOpen` (`text-primary`), leaves
`File` or a custom icon. Selected row = `bg-accent font-medium`.

### 14.15 Toast (`sonner.tsx` + `sonner`)
The `<Toaster position="bottom-right" richColors />` is mounted once in
`main.tsx` and themed from `resolvedMode`. Call imperatively: `toast(...)`,
`toast.success/error/info(...)`, `toast(msg, { action: { label, onClick } })`,
and `toast.promise(p, { loading, success, error })`. `richColors` gives
success/error/info their semantic tints.
